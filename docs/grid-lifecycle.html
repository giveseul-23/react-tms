1. 개요 — 그리드 컴포넌트 구성

DataGrid/index.tsx 본체는 직접 로직을 갖지 않고, 책임별로 분리된 훅들을 순서대로 호출해 <AgGridReact> 에 전달할 값을 조립합니다. ag-grid 는 명령형(imperative) 이라, React 의 선언형 state ↔ ag-grid 엔진 사이를 잇는 다리가 곧 이 훅들입니다.

DataGrid/ ├─ index.tsx 본체. 아래 훅들을 호출 + <AgGridReact> 렌더 ├─ useCellRangeSelection 셀 드래그 범위 선택 + Ctrl+C 복사 → columnOrderRef 반환 ├─ useActivePreset 탭/preset 분기 + 컬럼 가공/audit/합계행 → finalColumnDefs ├─ useRowLifecycle rowData/selection 변화를 ag-grid 에 동기화 (4 effect) ├─ useGridHandlers ag-grid 이벤트 콜백 5종 (state/ref 동기화) ├─ useGridProps 위 핸들러+설정을 commonGridProps 로 안정 조립 ├─ Pagination 페이지네이션 UI └─ (gridUtils/) ├─ useAutoSize onGridReady/onFirstDataRendered + 컬럼 자동너비 └─ autosize 텍스트 폭 측정 유틸 (calcOptimalWidths)

＊ 조회조건(SearchFilters)·메뉴권한은 그리드 바깥의 preset(MasterDetailPage 등)에서 useResolvedSearchMeta(menuCode) 로 처리됩니다. 그리드 자체의 생명주기와는 별개입니다.

3. 생명주기 (단계별)

mount 부터 unmount 까지 시간순. 각 단계에서 어떤 훅/effect 가 발화하는지 표시했습니다.

Render — 파생값 계산

useActivePreset 가 현재 탭/preset 기준 finalColumnDefs(가공·audit·합계행 포함)와 activeRowData 를 계산. 이 시점엔 아직 ag-grid 인스턴스가 없음.

onGridReady — 엔진 핸들 캡처

ag-grid 가 준비되면 useAutoSize.handleGridReady 가 gridApiRef.current = e.api 로 API 를 잡고, columnOrderRef 에 열 순서를 저장. rowData 가 이미 있으면 한 프레임 뒤 autosize 실행.

onFirstDataRendered — 최초 autosize

첫 데이터가 그려진 직후 handleFirstDataRendered 가 컬럼 너비를 내용에 맞게 1차 산출.

조회 결과 도착 — 데이터 set

Controller 가 model.grids.x.setData(obj) 로 객체를 직접 set → autoSizeKey +1 → useAutoSize effect 재발화(컬럼 재사이즈). rowData 가 ag-grid 로 흘러 행이 렌더됨.

선택 동기화

Controller 가 박은 selectedRow 를 useRowLifecycle 가 __rid__ 로 찾아 ag-grid 시각 선택 + ensureNodeVisible. (PK 가 있으면 이전 선택을 자동 복원)

사용자 상호작용

행 클릭 → handleRowClicked → cascade 조회 / 셀 편집 → handleCellValueChanged → EDIT_STS 트래킹 / 드래그 → useCellRangeSelection 범위 선택·복사.

행 추가

base.addRow 로 행 수 증가 → useRowLifecycle 가 마지막 신규행(EDIT_STS:"I")으로 스크롤하고 컬럼 defaultValue 를 자동 채움.

Unmount — 정리

useCellRangeSelection 의 DOM 리스너 해제 등 cleanup. 이후 API 호출은 api.isDestroyed?.() 가드로 차단.

핵심 포인트 — autoSizeKey. autosize 재실행은 activeRowData 가 아니라 autoSizeKey 에 묶여 있습니다. setData 가 객체로 직접 set 될 때만(=조회 결과 도착) +1 되고, 셀 편집/행 추가 같은 함수형 updater 호출에는 변하지 않습니다. → 편집 중 가로 스크롤·수동 폭이 유지됩니다.

4. 훅 레퍼런스 (상세)

useActivePreset

"지금 활성 탭/plain 그리드" 기준으로 입력값을 골라 가공한 파생값 묶음.

  • activeColumnDefs — 활성 컬럼 + audit 컬럼 자동 append(audit truthy 시 standardAudit).
  • finalColumnDefs — processColumnDef 로 가공한 최종 컬럼(Lang 번역·정렬·날짜/숫자 포맷·codeKey 렌더러·popup/date 위젯·편집모드 변환). ag-grid 에 실제 전달.
  • summaryRow — summable:true 컬럼 합계의 하단 고정 합계행.
  • effectiveRowKeys / effectiveAutoSelect — isPrimaryKey 컬럼에서 PK 추출 + 자동 첫행 선택 여부.

useAutoSize

컬럼 자동 너비 + ag-grid API 캡처. handleGridReady/handleFirstDataRendered 반환.

  • handleGridReady 가 gridApiRef·columnOrderRef 를 채움 — autosize 를 꺼도(disableAutoSize) 이 캡처는 유지.
  • 재실행 effect dep 은 autoSizeKey/activeTab 중심(조회 시에만 재사이즈).
  • 너비 계산은 calcOptimalWidths — 헤더/셀 텍스트 폭 + 패딩, MIN 80 ~ MAX 400 clamp.
주의. 이 호출을 통째로 주석 처리하면 autosize 뿐 아니라 gridApiRef 가 안 잡혀 선택·복사·라이프사이클·엑셀이 모두 깨집니다. autosize 만 끄려면 disableAutoSize prop 을 쓰세요.

useCellRangeSelection

컨테이너 div 에 마우스 핸들러를 붙여 셀 드래그 범위 선택 + Ctrl+C 복사. columnOrderRef(열 colId 순서) 반환.

  • 셀 좌표 ↔ colId 변환표로 columnOrderRef 사용. 드래그 시작 시 최신 컬럼 순서로 갱신.

useRowLifecycle

state(선택·데이터) 변화를 명령형 ag-grid 에 반영하는 4개 effect.

  • ① selectedRow → ag-grid 시각 선택(+스크롤). api 발 선택이라 핸들러가 무시 → 무한루프 없음.
  • ② 자동 첫행 — 이전 선택을 PK 로 복원(selectedRowRef 우선, 없으면 prevSelectedRef).
  • ③ 신규행(EDIT_STS:"I") 추가 시 마지막 행으로 스크롤.
  • ④ 신규행 빈 field 에 컬럼 defaultValue 자동 적용(check 는 미지정 시 "N"). 같은 prev 반환으로 무한루프 가드.

useGridHandlers

ag-grid 이벤트 → 앱 state/ref 동기화 콜백 5종. 모두 메모(참조 안정).

  • handleRowSelected — selectedRows 동기화(같은 선택이면 set 스킵 → 편집 중 cellEditor 보호) + prevSelectedRef 갱신.
  • handleRowClicked — 행 클릭 → onRowClicked. 체크박스/INPUT/Shift 클릭은 무시.
  • handleCellValueChanged — withRowStatusTracking 으로 EDIT_STS 자동 갱신 후 콜백.
  • handleSelectionChanged — 사용자 액션 source 만 외부 전파(api/rowDataChanged 등 자동 이벤트 무시).

useGridProps

위 핸들러 + 컬럼/선택/트리 설정을 commonGridProps 한 덩어리로 메모.

<AgGridReact {...commonGridProps} rowData={rowDataWithRid} />
  • getRowId: __rid__(트리는 제외 — getDataPath 가 id 결정), pinnedBottomRowData=합계행.
  • rowSelection(single/multi·헤더체크박스), defaultColDef, treeProps 각각 useMemo.
  • 맨 끝 ...gridOptions — 외부 오버라이드 escape hatch(최종 우선).
왜 이렇게 메모하나. commonGridProps 참조가 매 렌더 바뀌면 ag-grid 가 cellEditor 등 내부 상태를 reset 합니다. 그래서 조각마다 useMemo 로 참조를 고정 — 편집 중 셀이 튕기지 않도록.

5. ref 묶음

ref채우는 곳역할
gridApiRef ref onGridReady (= e.api) ag-grid Grid API 핸들. 행 redraw/선택/표시컬럼 등 모든 명령형 그리드 제어의 단일 진입점.
columnOrderRef ref onGridReady + 드래그 시작 시 현재 컬럼 순서(colId 배열). 셀 범위선택/복사가 "열 인덱스 ↔ colId" 변환에 사용.
gridContainerRef ref wrapper <div> 부착 이 그리드의 최외곽 DOM. 셀 드래그 마우스 이벤트 기준 + 권한 딤 오버레이의 relative 컨테이너.
prevSelectedRef ref handleRowClicked/Selected 직전에 선택된 행(row 데이터). 중복 선택 이벤트 거르기 + 자동 첫행 복원 비교.
selectedRowRef ref render 시 selectedRow 추적 현재 선택 행의 최신값. 자동 첫행 복원에서 prevSelectedRef 보다 우선.

＊ internalGridRef(AgGridReact 인스턴스)는 현재 미사용이라 제거됨 — 그리드 제어는 전부 gridApiRef 로.

6. 데이터 흐름 시나리오

① 조회 (검색 → 그리드 갱신)

SearchFilters 조회
  → Controller fetchFn → model.grids.main.setData(obj)   // 객체 직접 set
     → autoSizeKey +1                                     // useAutoSize 재발화
     → rowData 흐름 → ag-grid 행 렌더
  → onSearchCallback → onMainGridClick(rows[0])           // 첫 행 선택
     → useRowLifecycle ① selectedRow → 시각 선택

② 행 클릭 (master → detail cascade)

행 클릭 → handleRowClicked(e)
  → prevSelectedRef = e.data
  → activeOnRowClicked(e.data)                            // = Controller onMainGridClick
     → base.handleRowClick("main", row, [{ to:"detail", fetch }])
        → detail 그리드 setData → detail autoSizeKey +1

③ 셀 편집 → 저장

셀 편집 → handleCellValueChanged
  → withRowStatusTracking: 기존행 EDIT_STS="U" / 신규행 "I" 유지
  → (autoSizeKey 안 바뀜: 함수형 setter → 가로 스크롤·수동 폭 유지)
저장 → base.saveGrid: dirty 추출 + required/포맷 검증 + dsSave

④ 행 추가

base.addRow("main", newRow)   // EDIT_STS:"I" + __rid__ 자동
  → useRowLifecycle ③ 마지막 신규행으로 스크롤
  → useRowLifecycle ④ 컬럼 defaultValue 자동 채움

7. 주의점 / 함정

주제요점
참조 안정성 commonGridProps/finalColumnDefs 참조가 매 렌더 바뀌면 ag-grid 가 cellEditor·autosize 를 reset/재실행. deps 를 정확히 잡거나 ref 로 안정화할 것. (행 클릭 시 수동 폭이 되돌아가던 이슈가 이 계열)
autosize 끄기 useAutoSize 호출을 주석 처리하지 말 것(gridApiRef 캡처까지 사라짐). autosize 만 끄려면 disableAutoSize prop.
autoSizeKey 규칙 객체 직접 set(조회) 에만 +1. 함수형 updater(편집/행추가)는 불변 → 편집 중 재사이즈 안 함.
selection source 필터 handleSelectionChanged 는 사용자 액션만 전파. 자동 이벤트로 박아둔 선택을 덮어쓰지 않음.
TreeGrid 이름 컬럼 트리(이름) 컬럼은 기본 고정폭. 내용 맞춤이 필요하면 getNameText 로 autosize(들여쓰기·아이콘 반영).
__rid__ 모든 행에 자동 부여되는 내부 식별자. getRowId 가 이걸 써서 셀 편집으로 객체 참조가 바뀌어도 선택 유지.
