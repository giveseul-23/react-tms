📘 DEVELOPER GUIDE

화면 제작 가이드

이 프로젝트의 모든 업무 화면은 MVC 5-파일 패턴 + MasterDetailPage를 기반으로 제작합니다.
새 화면을 추가하거나 레이아웃을 변경할 때 이 문서를 참고하세요.

🗂 전체 구조

모든 업무 화면은 아래 5개 파일로 구성됩니다. 각 파일은 단일 책임 원칙에 따라 역할이 분리됩니다.

View
{화면명}.tsx

UI 렌더링만 담당. MasterDetailPage로 조립. 비즈니스 로직 없음.

Controller
{화면명}Controller.tsx

이벤트 핸들러 · API 호출. 액션 정의(useCallback).

Model
{화면명}Model.ts

화면 상태(useState/useRef) + 공통코드 codeMap.

Columns
{화면명}Columns.tsx

그리드 columnDefs. LBL_* 다국어 키 사용.

Api

패턴 G ⭐ 표준 base hook 패턴 — useBaseModel / useBaseController 예: LgstgrpOprConfigMst (4그리드 + cascade + 외부 탭 + 동기화)

모든 화면은 src/app/feature/ 의 base hook 두 개 를 기반으로 한다. boilerplate(state / ref / fetch / cascade / add / save / actions)는 base hook 이 흡수, 화면은 화면 고유 핸들러(onMainGridClick, onAddXxx, onSaveXxx)와 JSX 레이아웃만 작성.

핵심 컨벤션

  • Model.ts: useBaseModel<GridKey>(menuCode) 한 줄 + 화면 고유 state(codeMap, 외부 탭 등)
  • Controller.tsx: useBaseController<GridKey>({ model }) 호출 + 화면 고유 핸들러 / actions 정의
  • View.tsx: preset menuCode={MENU_CODE} + <DataGrid {...model.bind("gridKey")} /> spread
  • storageKey: menuCode 기반 자동 (MENU_FOO_BAR → foo-bar-outer/top/bottom)
  • searchRef / filtersRef: useBaseModel 이 자체 보유. View 에서 useRef 안 만든다.
  • useSearchMeta: preset 내부에서 자동. View 에서 호출 안 한다 (menuCode prop 만).
Model.ts
import { useMemo, useState, useEffect } from "react";
import { useBaseModel } from "@/app/feature/useBaseModel";
import { useCommonStores } from "@/hooks/useCommonStores";
import { featureApi } from "./FeatureApi";

// 그리드 이름 union — 오타 시 컴파일 에러 (model.grids.man → 에러)
export type GridKey = "main" | "sub01" | "sub02" | "sub03";

export function useFeatureModel(menuCode: string) {
  const base = useBaseModel<GridKey>(menuCode);

  // 외부 탭 (옵션) — 화면 고유 state
  const [tabs, setTabs] = useState([]);
  const [activeTab, setActiveTab] = useState("");
  useEffect(() => {
    featureApi.getTypeList().then((res) => { /* tabs 세팅 */ });
  }, []);

  // 공통코드 lookup → codeMap
  const { stores } = useCommonStores({ xxxTcd: { sqlProp: "CODE", keyParam: "XXX_TCD" } });
  const codeMap = useMemo(() => { /* stores → codeMap 변환 */ }, [stores]);

  return { ...base, tabs, activeTab, setActiveTab, codeMap };
}

export type FeatureModel = ReturnType<typeof useFeatureModel>;
Controller.tsx
import { useCallback, useMemo } from "react";
import { useBaseController } from "@/app/feature/useBaseController";
import { makeAddAction, makeSaveAction } from "@/app/components/grid/commonActions";
import { featureApi as api } from "./FeatureApi";
import type { FeatureModel, GridKey } from "./FeatureModel";

export function useFeatureController({ model }: { model: FeatureModel }) {
  const base = useBaseController<GridKey>({ model });

  // 메인 fetch (외부 탭 조건 합치는 케이스)
  const fetchList = useCallback((p) => api.getList({ ...p, TYPE: model.activeTab }), [model.activeTab]);

  // 메인 행 클릭 — sub01/sub03 cascade, sub02 alsoReset
  const onMainGridClick = useCallback((row) =>
    base.handleRowClick("main", row, [
      { to: "sub01", fetch: (r) => api.getDetailList({ CNFG_CD: r.CNFG_CD }) },
      { to: "sub03", fetch: (r) => api.getI18nList({ CNFG_CD: r.CNFG_CD }) },
    ], { alsoReset: ["sub02"] }), [base]);

  // 메인 조회 콜백 — 첫 행 자동 cascade
  const handleSearch = useCallback((data) => {
    model.grids.main.setData(data);
    onMainGridClick(data?.rows?.[0]);
  }, [model.grids.main, onMainGridClick]);

  // 행 추가 — 부모 검증 + base.addRow 한 줄 (EDIT_STS:"I" 자동)
  const onAddSub01 = useCallback(() => {
    const main = model.grids.main.selectedRef.current;
    if (!base.requireParentRow(main, "메인코드")) return;
    base.addRow("sub01", { CNFG_CD: main.CNFG_CD, CNFG_DTL_CD: "" });
  }, [model, base]);

  // 저장 — confirmOnDelete + cascadeFrom afterSave 한 줄
  const onSaveMain = useCallback(() =>
    base.saveGrid("main", api.save, {
      confirmOnDelete: "삭제된 항목이 있습니다. 계속 진행하시겠습니까?",
    }), [base]);

  const onSaveSub01 = useCallback(() =>
    base.saveGrid("sub01", api.saveDetail, {
      afterSave: { cascadeFrom: "main", fetch: (m) => api.getDetailList({ CNFG_CD: m.CNFG_CD }) },
    }), [base]);

  // 그리드별 actions — Controller 가 책임 (View 는 binding 만)
  const mainActions = useMemo(() => [
    makeAddAction({ onClick: onAddMain }),
    makeSaveAction({ onClick: onSaveMain }),
  ], [onAddMain, onSaveMain]);

  return { fetchList, handleSearch, onMainGridClick, mainActions, ... };
}
View.tsx
"use client";
import { MasterDetailPage } from "@/app/components/layout/presets/MasterDetailPage";
import { SplitPane } from "@/app/components/layout/SplitPane";
import DataGrid from "@/app/components/grid/DataGrid";

export const MENU_CODE = "MENU_XXX";  // View 가 source of truth — Api 가 import

export default function Feature() {
  const model = useFeatureModel(MENU_CODE);
  const ctrl = useFeatureController({ model });

  return (
    <MasterDetailPage
      menuCode={MENU_CODE}                           {/* useSearchMeta + Skeleton 자동 */}
      searchProps={{
        fetchFn: ctrl.fetchList,
        onSearch: ctrl.handleSearch,
        searchRef: model.searchRef,                {/* model 자체 보유 */}
        filtersRef: model.filtersRef,
        pageSize: model.pageSize,
        menuCode: MENU_CODE,
      }}
      direction="vertical"
      defaultSizes={[55, 45]}
      storageKey={model.storageKeys.outer}        {/* 자동 생성 */}
      master={
        <SplitPane direction="horizontal" storageKey={model.storageKeys.top}>
          <DataGrid
            {...model.bind("main")}                  {/* rowData/total/page/pageSize/audit/setRowData 자동 */}
            columnDefs={CONFIG_COLUMN_DEFS}
            onRowClicked={ctrl.onMainGridClick}
            actions={ctrl.mainActions}
          />
          <DataGrid {...model.bind("sub01")} columnDefs={DETAIL_COLUMN_DEFS} ... />
        </SplitPane>
      }
      detail={
        <SplitPane direction="horizontal" storageKey={model.storageKeys.bottom}>
          <DataGrid {...model.bind("sub03")} columnDefs={I18N_COLUMN_DEFS} />
          <DataGrid {...model.bind("sub02")} columnDefs={DETAIL_I18N_COLUMN_DEFS} />
        </SplitPane>
      }
    />
  );
}

base 헬퍼 빠른 참조 (Controller 에서 호출)

  • base.callAjax(promise, msg?) — API Promise 한 번 감쌈 (성공/에러 토스트)
  • base.alert(msg) / base.confirm(msg, onYes) — 다이얼로그
  • base.search(page?) — 메인 그리드 재조회
  • base.searchSub(gridKey, promise) — 임의 그리드에 결과 set
  • base.resetGrids([keys]) — 지정 그리드들 비우기
  • base.handleRowClick(gridKey, row, cascade?, opts?) — selection set + cascade reset/fetch
  • base.addRow(gridKey, newRow) — 그리드 끝 push (EDIT_STS:"I" 자동)
  • base.requireParentRow(row, label) — 부모 행 선택+저장 검증 + alert + boolean
  • base.saveGrid(gridKey, apiFn, opts?) — dirty 추출 + dsSave + 후처리. opts: beforeSave, confirmOnDelete, afterSave: "refresh"|"none"|함수|{cascadeFrom, fetch}

useBaseModel 자동 제공

  • 그리드별 data/rows/setData/ref (lazy 등록 — Proxy 기반)
  • 그리드별 selected/setSelected/selectedRef
  • searchRef / filtersRef (useRef 자체 생성)
  • pageSize / setPageSize
  • layout / setLayout (localStorage 자동 동기화)
  • storageKeys: { outer, top, bottom, layout } (menuCode → kebab-case prefix)
  • bind(gridKey) — DataGrid spread 객체 (rowData, totalCount, currentPage, pageSize, onPageSizeChange, onPageChange, audit:true, setRowData)
⚠️
순환 import 주의: MENU_CODE는 View 에서 export, Api 가 import. Controller/Model 도 필요 시 import 가능 (함수 본문에서만 사용 — module-init 시점 사용 X).

📝 파일별 작성 방법

View — {화면}.tsx

JSX/레이아웃만. 비즈니스 로직 없음.

  • 레이아웃은 MasterDetailPage(또는 그 변형)로 구성
  • 조회 조건 영역은 useSearchMeta(MENU_CODE) + searchProps.meta
  • 조회 trigger / 조회 조건 / 검색 영역 제외 키는 useRef로 보관
  • 레이아웃 토글("side" | "vertical")과 화면별 유일한 storageKey 지정

Columns — {화면}Columns.tsx

⚠️
setGridData 전달 컨벤션
  • 삭제 컬럼 + 추가 기능(useGridAdd) 둘 다 있으면 필수. 신규 행(EDIT_STS:"I") 삭제 시 배열에서 실제로 제거하려면 setter 필요.
  • 기존 행만 삭제(서버에서 받은 행) → markDelete가 객체에 직접 EDIT_STS:"D"를 마킹하므로 setter 없이도 작동.

실무 권장: 추가 기능이 붙을 가능성이 조금이라도 있으면 항상 MAIN_COLUMN_DEFS(model.setGridData)로 넘기세요. setter를 줘도 추가가 없으면 호출되지 않아 부작용 없음.

Api — {화면}Api.ts

  • 객체 형태로 export: export const featureApi = { MENU_CD, getList, save, ... }
  • 모든 요청에 MENU_CD 포함 + withSession()으로 세션 필드 주입
  • 저장 API는 dsSave 패턴: body에 { dsSave }, params에 세션/MENU_CD/그 외 키
  • 배열 페이로드일 때도 withSession이 각 원소에 세션 필드 자동 주입
api 골격
import { apiClient } from "@/app/http/client";
import { getSessionFields } from "@/app/services/auth/auth";

const MENU_CD = "MENU_XXX";

const withSession = (payload = {}) => {
  const s = getSessionFields();
  return Array.isArray(payload)
    ? payload.map((it) => ({ ...s, ...it }))
    : { ...s, ...payload };
};

export const featureApi = {
  MENU_CD,
  getList(payload) {
    return apiClient.post("/featureService/searchList",
      withSession({ MENU_CD, ...payload }));
  },
  save(payload) {
    return apiClient.post("/featureService/save",
      withSession(payload.dsSave));
  },
};
