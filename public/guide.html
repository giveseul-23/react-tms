📘 DEVELOPER GUIDE

화면 제작 가이드

이 프로젝트의 모든 업무 화면은 MVC 4-파일 패턴 + StandardPageLayout을 기반으로 제작합니다.
새 화면을 추가하거나 레이아웃을 변경할 때 이 문서를 참고하세요.

🗂 전체 구조

모든 업무 화면은 아래 4개 파일로 구성됩니다. 각 파일은 단일 책임 원칙에 따라 역할이 분리됩니다.

View
{화면명}.tsx

UI 렌더링만 담당. StandardPageLayout 으로 조립. 비즈니스 로직 없음.

Model
{화면명}Model.ts

상태(state / ref) 관리. 비즈니스 로직 없음.

Controller
{화면명}Controller.tsx

이벤트 핸들러 · API 호출 로직.

Columns
{화면명}Columns.tsx

그리드 컬럼 정의. Lang.get() 으로 헤더 다국어 처리.

bash 
src/views/{모듈명}/{화면명}/
├── {화면명}.tsx              ← View   : UI 렌더링만 담당
├── {화면명}Model.ts          ← Model  : 상태(state/ref) 관리
├── {화면명}Controller.tsx    ← Controller : 이벤트·API 호출 로직
└── {화면명}Columns.tsx       ← Columns : 그리드 컬럼 정의

⬜ 레이아웃 패턴 3가지

새 화면을 만들기 전에 아래 3가지 중 하나를 선택합니다.

패턴 A 단일 그리드 예시: 메뉴구성(MenuConfig)
조회조건 (SearchFilters)
그리드 1개 (DataGrid / TreeGrid)

→ singleGrid prop 전달. 레이아웃 토글 버튼 자동 숨김.

패턴 B 분할 그리드 예시: 운송사요청목록(TenderReceiveDispatch)
조회조건 (SearchFilters) + 레이아웃 토글 버튼 🔀
메인 그리드
서브 그리드 (탭 가능)

→ dualGrid prop 전달. layout 상태로 좌우 ↔ 상하 전환. 레이아웃 토글 버튼 자동 표시.

패턴 C 분할 그리드 + 하단 슬라이드 패널 예시: 운송사요청목록 추적 패널
조회조건 (SearchFilters) + 레이아웃 토글 버튼 🔀
메인 그리드
서브 그리드
하단 슬라이드 패널 (open 상태로 토글)

→ dualGrid + bottomPanel prop 전달. bottomPanel.open 으로 접기/펼치기 제어.

📝 파일별 작성 방법

Columns 파일

컬럼 정의만 모아두는 파일입니다. View / Controller 에서 import 해 사용합니다.

⚠️ headerName은 반드시 Lang.get('코드값') 형태로 작성합니다. 하드코딩 문자열 금지.
tsx — {화면명}Columns.tsx 
import { Lang } from "@/app/services/common/Lang";
import type { ColDef } from "ag-grid-community";

export const MAIN_COLUMN_DEFS = (): ColDef[] => [
  {
    headerName: Lang.get("COLUMN_CODE"),   // 언어팩 사용
    field: "CODE",
    width: 120,
  },
  {
    headerName: Lang.get("COLUMN_NAME"),
    field: "NAME",
    flex: 1,
  },
];

Model 파일

화면에서 사용하는 state 와 ref 만 정의합니다. 비즈니스 로직은 넣지 않습니다.

ts — {화면명}Model.ts 
import { useState, useRef, useCallback } from "react";
import { LayoutType } from "@/app/components/layout/LayoutToggleButton";

export function use{화면명}Model() {
  // ── 레이아웃 (분할 그리드가 있는 경우만)
  const [layout, setLayout] = useState<LayoutType>("side");

  // ── 페이징
  const [pageSize, setPageSize] = useState(20);

  // ── 메인 그리드 데이터
  const [gridData, setGridData] = useState({
    rows: [] as any[],
    totalCount: 0,
    page: 1,
    limit: 20,
  });

  // ── 선택 행 (ref 와 state 동시 관리)
  const [selectedRow, setSelectedRow] = useState<any>(null);
  const selectedRowRef = useRef<any>(null);
  const setSelectedRowWithRef = useCallback((row: any) => {
    setSelectedRow(row);
    selectedRowRef.current = row;
  }, []);

  return {
    layout, setLayout,
    pageSize, setPageSize,
    gridData, setGridData,
    selectedRow,
    setSelectedRow: setSelectedRowWithRef,
    selectedRowRef,
  };
}

export type {화면명}Model = ReturnType<typeof use{화면명}Model>;
💡 서브 그리드가 필요하면 subRowData, setSubRowData 상태를 추가합니다.
트리 그리드이면 rows 대신 source, setSource 상태를 사용합니다.

Controller 파일

API 호출과 이벤트 핸들러를 정의합니다.

tsx — {화면명}Controller.tsx 
import { useCallback, MutableRefObject } from "react";
import { useApiHandler } from "@/hooks/useApiHandler";
import { usePopup } from "@/app/components/popup/PopupContext";
import { {화면명}Model } from "./{화면명}Model";
import { {화면명}Api } from "@/features/{모듈}/{화면명}Api";

type ControllerProps = {
  model: {화면명}Model;
  searchRef: MutableRefObject<((page?: number) => void) | null>;
  filtersRef: MutableRefObject<Record<string, unknown>>;
};

export function use{화면명}Controller({ model, searchRef, filtersRef }: ControllerProps) {
  const { handleApi } = useApiHandler();
  const { openPopup, closePopup } = usePopup();

  // ── 조회 API — SearchFilters 의 fetchFn 으로 주입
  const fetchList = useCallback(
    (params: Record<string, unknown>) => {화면명}Api.getList(params),
    []
  );

  // ── 조회 결과 콜백 — SearchFilters 의 onSearch 으로 주입
  const handleSearch = useCallback(
    (result: any) => { model.setGridData(result); },
    [model]
  );

  // ── 행 클릭
  const handleRowClicked = useCallback(
    (row: any) => { model.setSelectedRow(row); },
    [model]
  );

  // ── 툴바 액션 목록
  const mainActions = [
    {
      type: "button" as const,
      key: "저장",
      label: "저장",
      onClick: (e: any) => {
        handleApi({화면명}Api.save(e.data), "저장되었습니다.")
          .then(() => searchRef.current?.());
      },
    },
  ];

  return { fetchList, handleSearch, handleRowClicked, mainActions };
}

View 파일 — 패턴 A (단일 그리드)

tsx — {화면명}.tsx 
"use client";

import { useRef } from "react";
import { Skeleton } from "@/app/components/ui/skeleton";
import { StandardPageLayout } from "@/app/components/layout/StandardPageLayout";
import DataGrid from "@/app/components/grid/DataGrid";
import { useSearchMeta } from "@/hooks/useSearchMeta";
import { use{화면명}Model } from "./{화면명}Model";
import { use{화면명}Controller } from "./{화면명}Controller";
import { MAIN_COLUMN_DEFS } from "./{화면명}Columns";

const MENU_CODE = "MENU_XXX_YYY";   // 서버 조회조건 메타 키

export default function {화면명}() {
  const searchRef = useRef<((page?: number) => void) | null>(null);
  const filtersRef = useRef<Record<string, unknown>>({});

  const { meta, loading } = useSearchMeta(MENU_CODE);
  const model = use{화면명}Model();
  const ctrl = use{화면명}Controller({ model, searchRef, filtersRef });

  if (loading) return <Skeleton className="h-24" />;

  return (
    <StandardPageLayout
      meta={meta}
      fetchFn={ctrl.fetchList}
      onSearch={ctrl.handleSearch}
      searchRef={searchRef}
      filtersRef={filtersRef}
      pageSize={model.pageSize}

      // singleGrid — 레이아웃 토글 버튼 자동 숨김
      singleGrid={{
        render: (
          <DataGrid
            layoutType="plain"
            columnDefs={MAIN_COLUMN_DEFS()}
            rowData={model.gridData.rows}
            totalCount={model.gridData.totalCount}
            currentPage={model.gridData.page}
            pageSize={model.pageSize}
            onPageSizeChange={model.setPageSize}
            onPageChange={(page) => searchRef.current?.(page, false)}
            actions={ctrl.mainActions}
            onRowClicked={ctrl.handleRowClicked}
          />
        ),
      }}
    />
  );
}

DataGrid 대신 TreeGrid를 사용하는 경우 추가 작업이 필요합니다.

Controller handleSearch

ts 
const handleSearch = useCallback((result: any) => {
  const list = result?.rows ?? [];

  // DYNAMIC_QUERY 가 "1=1" 이면 전체조회, 그 외엔 필터링 조회
  const dynamicQuery = String(
    result?.DYNAMIC_QUERY ?? filtersRef.current?.DYNAMIC_QUERY ?? "1=1"
  ).trim();
  const isFiltered = dynamicQuery !== "1=1";

  // buildSource: isFiltered=true 이면 자식 없는 루트 노드 제거
  model.setSource(buildSource(list, isFiltered));
}, [model, filtersRef]);
💡 buildSource(list, isFiltered) — 두 번째 인자가 true이면 data[]가 비어 있는 가상 루트(-1) 노드를 트리에서 자동 제외합니다. 전체 조회(false)일 때는 빈 루트도 표시합니다.
